# SiteGround PHP Mail Form — Blueprint

> Baseline documentation for self-hosted contact forms on SiteGround shared hosting.  
> Reusable for: **SevenRoofing**, **SmartDoorsAndWindows**, **SevenRoofingNorth**, and any future static sites.

---

## Overview

This approach replaces third-party form services (Web3Forms, Formspree, EmailJS, etc.) with a single PHP file that uses SiteGround's built-in `mail()` function. No external APIs, no npm packages, no SMTP credentials needed.

### Architecture

```
Browser (HTML form)
    ↓ POST /contact.php
SiteGround PHP (mail())
    ↓ SMTP via SiteGround's mail server
Inbox (info@yourdomain.com.au)
```

### Why This Approach

| Factor | Web3Forms / 3rd-party | SiteGround PHP mail() |
|--------|----------------------|----------------------|
| Cost | Free tier limits, paid plans | Free (included with hosting) |
| Privacy | Data passes through external servers | Data stays on your server |
| Reliability | Depends on 3rd-party uptime | Depends on your hosting (99.9% SLA) |
| Setup | API key, external account | Single PHP file |
| Email headers | Generic sender | Custom From/Reply-To matching your domain |
| Dependencies | JS fetch + external API | Zero — PHP only |

## Implementation Guide

### 1. Create `contact.php`

Place this at your project root. The build script will copy it to the output folder.

```php
<?php
// Only accept POST requests
if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    header('Location: /contact/');
    exit;
}

// Honeypot — if a bot fills this hidden field, silently discard
if (!empty($_POST['website'])) {
    header('Location: /thank-you/');
    exit;
}

// Sanitize all inputs
$name           = htmlspecialchars(trim($_POST['name'] ?? ''), ENT_QUOTES, 'UTF-8');
$email          = filter_var(trim($_POST['email'] ?? ''), FILTER_SANITIZE_EMAIL);
$phone          = htmlspecialchars(trim($_POST['phone'] ?? ''), ENT_QUOTES, 'UTF-8');
$address        = htmlspecialchars(trim($_POST['address'] ?? ''), ENT_QUOTES, 'UTF-8');
$message        = htmlspecialchars(trim($_POST['message'] ?? ''), ENT_QUOTES, 'UTF-8');
$page           = htmlspecialchars(trim($_POST['page'] ?? ''), ENT_QUOTES, 'UTF-8');
$redirect_error = filter_var(trim($_POST['_redirect_error'] ?? '/contact/?error=1'), FILTER_SANITIZE_URL);

// Validate required fields (name + valid email only; message is optional)
if (empty($name) || !filter_var($email, FILTER_VALIDATE_EMAIL)) {
    header('Location: ' . $redirect_error);
    exit;
}

// Build email
$to      = 'info@yourdomain.com.au';              // ← CHANGE THIS
$subject = 'Enquiry From ' . $name;

$body  = "You have received a new enquiry from the website.\n\n";
$body .= "Name: {$name}\n";
$body .= "Email: {$email}\n";
if ($phone)   $body .= "Phone: {$phone}\n";
if ($address) $body .= "Address: {$address}\n";
if ($message) $body .= "\nMessage:\n{$message}\n";
if ($page)    $body .= "\n---\nSent from: https://yourdomain.com.au{$page}\n";

// Headers
$headers  = "From: YourBrand <no-reply@yourdomain.com.au>\r\n";  // ← CHANGE THIS. 
$headers .= "Reply-To: {$email}\r\n";
$headers .= "Content-Type: text/plain; charset=UTF-8\r\n";
$headers .= "X-Mailer: PHP/" . phpversion();

// Send email — use 4-parameter form only.
// IMPORTANT: Do NOT use the 5th parameter (-f flag) on SiteGround shared hosting.
// It silently causes mail delivery to fail.
mail($to, $subject, $body, $headers);

// Always redirect to thank-you.
// mail() on SiteGround may return false even when the email is queued successfully,
// so we do not check its return value.
header('Location: /thank-you/');
exit;
```

### 3. Client-Side JavaScript (`form-handler.js`)

Handles error display and analytics tracking. The form submits natively to PHP — no AJAX needed.

```javascript
document.addEventListener('DOMContentLoaded', function() {
    // Show error message if redirected back with ?error=1
    if (window.location.search.indexOf('error=1') !== -1) {
        var forms = document.querySelectorAll('.enquiry-form');
        forms.forEach(function(form) {
            var status = form.querySelector('.form-status');
            if (status) {
                status.textContent = 'Sorry, there was an error. Please try again or call us.';
                status.className = 'form-status error';
            }
        });
    }

    // Fire analytics before native form submission
    var forms = document.querySelectorAll('.enquiry-form');
    forms.forEach(function(form) {
        form.addEventListener('submit', function() {
            var btn = form.querySelector('button[type="submit"]');
            if (btn) {
                btn.textContent = 'Sending...';
                // Do NOT set btn.disabled = true — can prevent native POST in some browsers
            }
            window.dataLayer = window.dataLayer || [];
            dataLayer.push({
                event: 'form_submit',
                form_name: form.id || 'enquiry_form',
                form_location: window.location.pathname
            });
            if (typeof gtag === 'function') {
                gtag('event', 'generate_lead', {
                    event_category: 'form',
                    form_name: form.id || 'enquiry_form',
                    page_location: window.location.href,
                    transport_type: 'beacon'
                });
            }
        });
    });
});
```

Add these two functions to your `build-site.js` and call them **after** all pages are built:

function copyContactPhp() {
  const src = path.join(__dirname, 'contact.php');
  const dest = path.join(BUILD, 'contact.php');
  if (fs.existsSync(src)) {
    fs.copyFileSync(src, dest);

### 5. Thank-You Page

Generated by the build script at `/thank-you/index.html` by cloning the contact page layout.

**What's included:**
- Full site header with navigation (all menu links work)
- Inner banner with "Thank You" heading
- Professional success section: checkmark icon, confirmation message, "Back to Home" + "Call Us" buttons
- Brands carousel section
- Client reviews section
- Full site footer with quick links, services, contact info
- GA4 conversion event firing on page load
- `noindex` meta tag (should not appear in search)

**What's excluded:**
- `pages/contact.css` (not needed)
- `form-handler.js` (no forms on this page)
- Schema JSON-LD, OG/Twitter meta tags
- Contact page "active" state in navigation

This approach ensures the user experiences a seamless transition — the thank-you page looks and feels identical to the rest of the site, with full navigation to continue browsing.

## Security Features

| Feature | Implementation |
|---------|---------------|
| **Honeypot** | Hidden `website` field — bots fill it, humans don't. Silently redirects to thank-you. |
| **Input sanitization** | `htmlspecialchars()` on all text, `FILTER_SANITIZE_EMAIL` on email |
| **Email validation** | `FILTER_VALIDATE_EMAIL` — rejects malformed emails |
| **POST-only** | GET requests redirect to /contact/ |
| **No header injection** | Email address validated before use in Reply-To header |
| **CSRF-lite** | `_redirect_error` field is sanitized with `FILTER_SANITIZE_URL` |

## Deployment Checklist

### Before First Deploy

- [ ] Verify PHP is enabled on SiteGround (it is by default on all shared plans)
- [ ] Verify `mail()` function is not disabled — check PHP Settings in Site Tools
- [ ] Set up email account `no-reply@yourdomain.com.au` in SiteGround Email > Accounts (optional but improves deliverability)
- [ ] Verify SPF record includes SiteGround's mail server (usually auto-configured)

### After Deploy

- [ ] Test form submission from the live site
- [ ] Check email arrives at `info@yourdomain.com.au`
- [ ] Verify Reply-To works (reply to the test email, should go to submitted address)
- [ ] Verify `Subject: Enquiry From [Name]` format
- [ ] Test honeypot by submitting with `website` field filled (use browser dev tools)
- [ ] Test validation by submitting without required fields
- [ ] Verify error redirect works (should show error message on same page)
- [ ] Verify thank-you page loads at `/thank-you/`
- [ ] Check GA4 real-time reports for `form_submit` and `conversion` events

### SiteGround-Specific Notes

- SiteGround uses `gsydmXXXX.siteground.biz` as the mail relay — this appears in email headers as "via" and "mailed-by"
- PHP `mail()` works out of the box on all SiteGround plans (GrowBig, GoGeek, etc.)
- Rate limit: SiteGround allows ~500 emails/hour on shared hosting — more than enough for contact forms
- If emails go to spam, add an SPF record: `v=spf1 include:_spf.siteground.biz ~all`

## Adapting for a New Project

### Quick Setup (copy-paste)

1. Copy `contact.php` to new project root
2. Edit these values in `contact.php`:
   - `$to` — destination email address
   - `From:` header — brand name and no-reply address
   - `$body` — the "Sent from" URL
3. Add the honeypot + hidden fields to your HTML forms
4. Add `form-handler.js` to your JS assets
5. Generate or create a thank-you page
6. If using a build script, add `copyContactPhp()` step

### Customization Points

| What to change | Where |
|---------------|-------|
| Recipient email | `$to` in contact.php |
| From name/address | `From:` header in contact.php |
| Required fields | Validation block in contact.php + `required` attribute in HTML |
| Form fields | Add fields to HTML + read them in contact.php |
| Success redirect | `/thank-you/` in contact.php |
| Error redirect | `_redirect_error` hidden field in HTML |
| Email subject | `$subject` in contact.php |
| GTM container ID | GTM script in thank-you page |
| GA4 measurement ID | gtag config in form-handler.js / thank-you page |

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Email not arriving | Check SiteGround Site Tools > Email > check if `mail()` is enabled. Test with a simple `<?php mail('you@email.com','Test','Body'); ?>` |
| Email not arriving (was working before) | **Do NOT use the 5th parameter** (`-f` flag) in `mail()` on SiteGround shared hosting. It silently prevents delivery. Use only the 4-parameter form: `mail($to, $subject, $body, $headers)` |
| Email goes to spam | Add SPF record. Ensure `no-reply@yourdomain` mailbox exists. Check email content for spam triggers. |
| Form shows error after submit | Check PHP error logs in SiteGround Site Tools > Statistics > Error Log. Also clear browser cache — old JS may be cached. |
| Form POST not working | Do **not** set `btn.disabled = true` on the submit button during the `submit` event — some browsers cancel the native POST if the button is disabled before the form submits. Only change button text. |
| Redirect loops | Ensure `_redirect_error` URL doesn't trigger another POST |
| mail() returns false but email arrives | This is normal on SiteGround. The function returns false even when the email is queued. Do not check the return value — always redirect to `/thank-you/`. |
| Honeypot blocking real users | Verify the hidden field has `style="display:none"` and `tabindex="-1"` |
| Analytics not firing | Check GTM/GA4 container IDs. Use `transport_type: 'beacon'` for pre-navigation events. |
